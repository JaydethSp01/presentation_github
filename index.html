01 — Git

¿Qué es Git?

Git es un sistema de control de versiones DISTRIBUIDO, creado en 2005 por Linus Torvalds para gestionar el kernel de Linux.

  • Funciona 100% localmente. Sin internet puedes hacer commits, ver historial y crear ramas.
  • Cada desarrollador tiene una copia completa del repositorio con todo el historial.
  • Usa funciones hash SHA-1 para garantizar que el historial nunca se corrompa.
  • No guarda diferencias entre archivos — guarda snapshots completos del proyecto.
Concepto clave: Git no necesita un servidor central. Si GitHub desaparece mañana, tu historial completo sigue viviendo en tu máquina.
2005
Año de Creación

Linus Torvalds lo construyó en 10 días para reemplazar el sistema que usaba el kernel de Linux.

01 — Git

Características de Git

🌿

Ramas (Branches)

Líneas paralelas de desarrollo. Puedes tener una rama main estable y una feature/login experimental al mismo tiempo.

📸

Commits = Snapshots

Cada commit guarda el estado completo del proyecto con autor, fecha y mensaje. Puedes volver a cualquier punto de la historia.

🔀

Merge inteligente

Git une cambios de ramas distintas automáticamente. Si hay conflictos, te avisa exactamente en qué línea y te deja resolverlos sin perder nada.

⚡

Alta velocidad

Commit, branch, merge — todo ocurre localmente, sin llamadas de red. Git comprime los datos eficientemente con packfiles.

🔐

Integridad SHA

Cada objeto en Git tiene un hash único. Es imposible modificar el historial sin que Git lo detecte.

📴

Trabajo offline

No necesitas internet para hacer commit, revisar log o moverte entre ramas. Sincronizas cuando quieras.

02 — GitHub

¿Qué es GitHub?

100M+
Desarrolladores

La plataforma de código más grande del mundo. Adquirida por Microsoft en 2018 por $7.5 mil millones.

GitHub NO es Git. Es una plataforma en la nube construida sobre Git. Lanzada en 2008, es la red social de los desarrolladores.

  • Alojamiento de repositorios en la nube
  • Pull Requests — revisión formal de código antes de integrar
  • Issues — registro de bugs, tareas y mejoras
  • GitHub Actions — automatización de tests y despliegues
  • GitHub Pages — hosting gratuito de sitios estáticos
  • GitHub Copilot — asistente de IA para escribir código
03 — Diferencias

Git vs GitHub

Git

La herramienta. La navaja.

  • Software instalado en tu PC
  • Funciona sin internet (local)
  • Interfaz de línea de comandos
  • 100% gratuito y open source
  • Gestiona versiones del código
  • Creado en 2005 por Torvalds
VS

GitHub

El ecosistema. La plataforma.

  • Servicio web en la nube
  • Requiere conexión a internet
  • Interfaz gráfica web + CLI
  • Plan gratuito + planes de pago
  • Colaboración, CI/CD, hosting
  • Fundado en 2008, comprado por Microsoft
Analogía: Git es el motor de un carro. GitHub es la autopista, las gasolineras y el GPS. Puedes tener el motor sin la autopista, pero no la autopista sin el motor.
04 — Repositorio

El Repositorio de Versiones

Un repositorio es la carpeta donde Git vigila absolutamente todo. Contiene el historial completo del proyecto desde el primer commit.

  • Commits: Cada guardado tiene hash único, autor, fecha y mensaje descriptivo.
  • Branches: Ramas paralelas — main, develop, feature/x.
  • Tags: Marcadores en puntos importantes. Ej: v1.0.0.
  • README.md: La carta de presentación del proyecto.
  • .gitignore: Lista de lo que Git NO debe rastrear (contraseñas, node_modules, builds).

06 — Práctica

Flujo Completo: Pasos 1–3

Creamos una app en Node.js desde cero y la subimos a GitHub. Proyecto real, comandos reales.

1

Configuración inicial (una sola vez)

bash
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
2

Crear el repositorio local

bash
git init mi-app-node   # Crea la carpeta y activa Git
cd mi-app-node
npm init -y          # Genera package.json automáticamente
3

Primer commit (primer snapshot)

bash
git add .                 # "." = agrega TODOS los archivos al staging
git commit -m "feat: inicializar proyecto Node.js"
06 — Práctica

Flujo Completo: Pasos 4–6

4

Conectar y subir a GitHub

bash
git remote add origin https://github.com/usuario/mi-app.git
git branch -M main    # Renombra rama a 'main'
git push -u origin main  # -u enlaza local con remoto
5

Trabajar en una nueva rama (sin romper main)

bash
git checkout -b feature/login  # Crea y cambia a nueva rama
# ... escribes el módulo de login ...
git add . && git commit -m "feat: módulo autenticación"
git push origin feature/login
6

Pull Request en GitHub → revisión → merge

En GitHub vas a tu repositorio, aparece un botón "Compare & pull request". Describes los cambios, un compañero los revisa, y al aprobar se hace merge a main. El código nunca llega sucio a producción.
07 — CI/CD

Integración y Despliegue Continuo

En términos simples: cada vez que haces git push, GitHub ejecuta servidores virtuales automáticamente para testear y desplegar tu aplicación.

  • Integración Continua (CI): Ejecuta tests automáticos en cada push. Si algo falla, el equipo lo sabe antes de llegar a producción.
  • Despliegue Continuo (CD): Si los tests pasan, el código se despliega solo al servidor. Sin intervención manual.
  • GitHub Pages: Hosting gratuito para sitios estáticos. Esta presentación puede vivir ahí.
Los workflows se configuran con archivos .yml dentro de .github/workflows/
.github/workflows/deploy.yml
name: CI/CD Pipeline
on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - run: npm install
      - run: npm test # ← Tests automáticos

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
08 — Análisis

Ventajas & Desventajas

✔ Ventajas de GitHub

  • Comunidad global — más de 100M de devs y millones de proyectos open source.
  • GitHub Actions — CI/CD sin salir de la plataforma, con miles de actions reutilizables.
  • GitHub Pages — hosting estático gratuito desde tu repositorio.
  • Copilot — IA integrada que sugiere código en tiempo real.
  • Issues + Projects — gestión de tareas directamente en el repo.
  • Plan gratuito generoso — repositorios públicos y privados ilimitados.

✘ Desventajas de GitHub

  • ▸ Dependencia de internet — si GitHub cae (y ha caído), el equipo se detiene.
  • ▸ Privacidad en manos de Microsoft — tu código vive en servidores de terceros.
  • ▸ Curva de aprendizaje — Git + GitHub juntos pueden abrumar a principiantes.
  • ▸ Actions avanzados de pago — minutos de CI/CD se agotan en proyectos grandes.
  • ▸ Conflictos de merge — en equipos grandes sin buenas prácticas son frecuentes.
